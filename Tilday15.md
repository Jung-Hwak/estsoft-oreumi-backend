## Java

### 메서드

* main 메서드 안에서 모든 기능을 작성하면 프로그램은 난잡해지고 복잡해진다.
1. 함수(functions)
   - 특정한 기능을 수행하는 프로그램 코드를 모아놓은 것
   - 한 번 작성해 놓고 내가 필요할때 불러와 쓸 수 있다.
2. 메서드의 정의
   - 메서드를 정의할 때는 메서드 원형과 메서드 본체를 작성한다.
3. 메서드의 호출 
   - 메서드를 호출하게 되면 호출한 메서드의 본체를 실행한다.
   - 메서드를 호출했을 때 프로그램의 실행 흐름은 메서드 정의로 이동하게 된다.
   - 메서드 본체의 실행이 끝나면, 프로그램 실행 흐름은 메서드 호출 표현식으로 되돌아온다.
4. 메서드의 매개 변수
   - 메서드가 지정된 기능을 수행하는데 필요한 값을 전달받기 위한 변수이다.
   - 메서드를 정의하면서 메서드 식별자 다음에 나오는 괄호안에 쉼표(,)로 구분해서 나열한다.
   - 매개 변수는 메서드를 호출하면 선언되고, 인자로 초기화된다.
5. 메서드의 반환값
   - 메서드에서 return 문으로 실행 결과를 메서드 호출 표현식에 전달(반환)할 수 있다.
   - 메서드가 반환하는 값을 오로지 하나이며, 메서드 호출 표현식의 연산 결과가 된다.
   - 메서드의 반환값이 없으면 반환 값 자료형을 void로 한다. 이때는 return 문은 생략 할 수 있다.
6. 인자
   - 메서드를 호출하면서, 메서드에 전달하는 값이다.
   - 메서드 호출 연선자에 쉽표(,)로 구분해서 나열한다.
7. 지역 변수
   - 블록 내에서 선언한 변수로, 선언된 블록 내에서만 접근할수 있는 변수이다. 
   - 메서드가 호출되어야 선언이되며, 변수가 선언된 블록이 종료되면 저장공간이 소멸한다.
   - 메서드의 매개 변수도 지역 변수이다.
8. 인자
   - 메서드를 호출하면서 지정해주는, 메서드에서 전달하는 값을 인자 라고한다.
   - 매게 변수가 있는 메서드를 호출할 때는 매게 변수의 개수와 자료형에 맞게 인자를 지정해줘야 한다.
   - 인자가 매개변수의 개수와 자료형이 맞지않으면 컴파일 오류가 발생한다.

### 절차 지향, 객체 지향 프로그래밍

1. 절차 지향 프로그래밍(Procedural Programming)
   - 어떤 문제를 해결해 가는데 에 대해 한문장씩 해결하는 방법이다.
   - 프로그램의 규모가 커지면 코드 관리 및 유지 보수에 어려움이 생긴다.
2. 객체 지향 프로그래밍(Object-Oriented Programming)
   - 프로그램을 명령들의 흐름으로 보는게 아닌 객체들의 집합으로 본다.
   - 데이터와 절차를 하나의 객체로 묶어서 생각하는 방법이다.
   - 캡슐화, 상속, 다형성 등을 통해 코드의 관리와 유지 보수가 쉬워진다.
   1. 자료 추상화
      -  특정 대상을 표현하는 여러 값과 기능을 하나로 묶어서 정리한 것
      -  캡슐화를 통해 외부로부터 객체의 정보를 숨길 수 있다.
   2. 상속
      - 이전에 작성한 클래스를 이어받아 새로운 클래스를 생성하는 기법이다.
      - 기존의 코드를 재활용 하기 위한 방법이다.
   3. 다형성
      - 이름 하나로 여러 상황에 대처하는 기법이다.
      - 개념적으로 동일한 작업을 하는 메서드들에 같은 이름을 부여할 수 있으므로 코드가 더 간단해진다. 

### 클래스와 객체
    클래스의 예시: 자동차
        ```
        class car {
            String color;
            int speed;
            int gear;

            void print(){
                System.out.println("Color: " + color);
                System.out.println("Speed: " + speed);
                System.out.println("Gear" + gear);
            }
        } 
             ↓ Car 자료형의 참조 변수 선언이다.
        Car myCar = new Car();
                     ↑ new연산자로 객체를 생성한다.
1. 클래스 
   - 객체를 만들기 위한 자료형이 클래스이다.
   - 어떤 클래스를 가지고서 생성한 객체를 그 클래스를 인스턴스라고 한다.
   - 자료형이 클래스인 변수들을 우리는 객체라고 부른다.
2. 객체
   - 객체는 속성(attributes)과 기능(methods)의 집합이며, 객체에 속한 속성과 기능을 객체의 멤버(member; 구성요소)라고 한다.
   - Java 에서는 속성은 필드(fields)로 정의하며, 기능은 메서드(methods)로 정의한다.
   - 클래스를 정의를 해야 객체를 생성할 수 있다.
   - 객체를 생성할때는 배열을 생성할때 쓰는 new 연산자를 사용하여 객체를 생성한다.

### 생성자

1. 생성자
   - 객체가 생성될 때 호출되는 "인스턴스 초기화 메서드"
   - 일반적으로 필드를 초기화하는 열학을 수행한다.
   - 생성자의 식별자는 클래스의 식별자와 같아야한다.
   - 생성자를 호출할 수 없으 면 객체는 생성할 수 없다.
   - 인스턴스는 클래스 외부에서 생성되므로 접근 제어 지시자를 "public"으로 설정한다.
   * 생성자 오버로딩
     - 생성자도 메서드 이므로, 메서드 오버로딩을 할 수 있다.
     - 메서드 오버로딩은 매개 변수의 자료형이나 개수가 다르면, 식별자가 같은 메서드가 여러 개 있을 수 있다.
        ```
        public Sample(int n) { data = n;} 
        public Sample() { data = 0;} 
        Sample sample1 = new Sample(10)  // Sample#Sample(int)
        Sample sample2 = new Sample()  // Sample#Sample()
        객체 sample1이 생설될 때는 Sample(int) 생성자가 호출되고, sample2가 생성될 때는 Sample() 생성자가 호출된다.
        기본 생성자는 클래스에 생성자가 하나도 없어야 추가된다. 클래스에 생성자를 추가하면, 기본 생성자도 추가해야 한다.
2. 기본 생성자 
   - 클래스에 생성자가 없더라도 객체는 생성되어야 하므로, 컴파일러가 추가하는 생성자이다.
   - 클래스에 생성자가 있으면 기본 생성자는 추가되지 않는다.
   - 필드를 초기화하기 위해 생성자를 추가한 경우, 기본생성자는 추가되지 않기 때문에 기본 생성자의 역할을 할 생성자, 즉 매개 변수가 없는 생성자의 정의를
3. 생성자와 설정자의 매개 변수 식별자
  -  생성자의 매개 변수는 필드를 초기화할 값을 전달받기 위해 사용하고, 설정자의 매개 변수는 필드에 설정할 값을 전달받기 위해 사용한다.
  -  그래서 생성자와 설정자의 매개 변수는 필드의 식별자와 같은 식별자를 사용해서, 이 매개 변수에 전달된 값이 어떤 필드에 초기화 되는지 또는 설정되는지 명시하는게 좋다.
  * 식별자의 충돌
    ```
    public Sample(int data) { data = data; }
    public void setData(int data) { data = data; }

-  필드와 지역 변수의 식별자가 같으면, 지역 변수가 우선한다.
-  그래서 위 예에서 메서드 본체에서 사용한 식별자는 매게 변수의 식별자가 된다.

## 캡슐화와 정보 은닉

1. 캡슐화
   - 객체 지향 프로그램에서는 관련 있는 데이터와 기능을 묶어서 하나의 "블랙박스"를 생성하는데 이를 "캡슐화"라고 한다.
   - 객체는 독립적으로 완벽한 역할을 수행할 수 있어야 한다.
   - 객체를 사용하는 개발자는 객체의 정보와 기능을 활용할 수 있는 인터페이스만 알면 되며, 내부 동작은 알 필요가 없다.
2. 정보 은닉의 필요성
   - 의도치 않게 외부에서 접근해서 객체가 가지고 있는 데이터를 오염시킬 수 있다. 그로인해 오류가 발생하는걸 방지하기 위해, 객체가 가지고 있는 정보를 외부에서는 보이지 않게 숨긴다.
   - 이를 위해 클래스에서 필드는 일반적으로 `private` 멤버로 선언한다
   1. 접근 제어
      - 객체의 멤버에 접근하는것을 제어하는 것
        ```
                     /\ 
        private      || 전용 멤버: 객체 내부에서만 접근 가능
        default      || 패키지 멤버: 패키지 내부에서만 접근 가능
        protected    || 보호 멤버: 객체 내부 또는 상속받은 클래스의 객체에서만 접근가능
        public       || 공용 멤버: 어디에서나 접근 가능
    - 접근 제어 지시자(Access specifiers)
      -  객체의 멤버에게 접근할 수 있는 범위를 설정한다.
      -  클래스를 정의 하면서 각가의 멤버 앞에 지정한다.
      -  접근 제어 지시자를 따로 지정하지 않으면 default로 설정이 된다.
    - 접근 제어의 종류
      - 클래스 수준에서의 접근제어
        - public클래스 : 다른 모든 클래스가 접근할 수 있는 공용 클래스
        - 아무런 지시자도 없는 경우, 같은 패키지 안에 있는 클래스에서만 접근할 수 있다.
    - 멤버 수준에서의 접근 제어
      - 클래스를 정의하면서 멤버 앞에 접근 제어 지시자로 접근 범위를 설정한다.
      - 접근 제어 지시자를 따로 지정하지 않으면 default가 적용돼 패키지 멤버가 된다.
### 변수의 종류

1. 필드(Fields)              : 클래스에서 정의한 멤버 변수(인스턴스 변수)
    - 필드는 각각의 객체들 마다 개별적인 저장 공간이 할당이 된다.
    - 필드가 포함 되어있는 객체는 참조 변수가 없으면 가비지 컬렉터에 의해 자동으로 제거된다.
3. 지역 변수(Local variables) : 메서드나 블록 안에서 선언한 변수
   - 메서드 안에서 선언되고, 메서드가 호출이 되어야 선언 된다.
   - 블록을 벗어나면 지역 변수들은 소멸하게 된다.
4. 매개 변수(Parameters)      : 메서드에 전달되는 값을 저장하기 위한 변수

### 설정자와 접근자

1. 설정자
   -  외부에서 접근 하지 못하도록 private 멤버로 선언한 필드를 설정하는 메서드이다.
   -  필드에 잘못된 값이 대입되는 것을 방지하기 위해 사용한다
   -  set... 형식으로 식별자를 지정해준다.
2. 접근자
   - private 멤버로 선언한 필드를 '참조'하는 메서드 이다.
   - 필드의 값을 계산해서 구해야 하는 경우라면, 접근자에서 계산한 다음 반환한다.
   - get... 형식으로 식별자를 지정해준다.
* 설정자(Setters)와 접근자(Getters)
  - 객체가 가진 값을 보호하기 위해 일반적으로 필드는 private 멤버로 선언한다.
  - 필드를 private 멤버로 선언하면 외부에서는 이필드에 접근할수 없다, 그래서 외부에서 이 필드에 접근할 수 있도록 설정자와 접근자를 제공한다.
  - 설정자는 필드에 값을 설정하기 전에 그 값을 검사한다. 그렇게 해서 필드에 잘못된 값이 설정되는 것을 방지할 수 있다.
  - 계산해서 값을 구해야 하는 필드는 접근자에서 값을 계산한 다음, 반환한다.
  * private 멤버로 선언한 모든 필드는 설정자와 접근자를 가져야 한다. 
    ```
    public void setData(int n) { if (n >= 0) data = n; } 
    public int getData() { return data; }
### this 참조 변수
* this
    - this 메서드
      - 생성자에서 다른 생성자를 호출하기 위한 메서드
      - this 메서드는 생성자의 첫 번째 문장에서만 호출할 수 있다.
      - 주어진 인자의 자료형과 개수에 따라 어떤 생성자가 호출될지 결정이된다.
        ```
        public Sample(int data) { this.data = data; } 
        public Sample() { this(0); }
        생성자의 역할은 객체를 초기화하는 것이다. 특히 필드를 초기화하는 것이 중요한 역할이다.
### 생성자

1. 복사 생성자
   - 객체를 생성하면서 기존 객체로 초기화하는 경우도 있다.
   - 이를 위해 클래스를 정의하면서 복사 생성자는 정의하도록 한다.
   ```
   public Sample(int data) { this.data = data; } 
   public Sample() { this(0); }
   public Sample(Sample copy) { this(copy.data); }
  ```
