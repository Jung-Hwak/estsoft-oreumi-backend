# 자바 웹 개발 수업 정리 (2026-01-08)

## 1. 웹 개발 환경 및 배포 자동화
현대적인 웹 개발에서는 설정 파일의 직접 수정을 지양하고, IDE와 빌드 도구를 활용한 안전한 배포 방식을 지향합니다.

* **서버 운영 방식의 변화**: 과거에는 설정 파일을 직접 건드렸으나, 현재는 훨씬 안전한 서비스 운영을 위해 IDE를 활용함.
* **WAR(Web Application Archive) 배포**:
    * 개발 결과물을 `.war` 파일로 압축하여 배포함.
    * 톰캣(Tomcat) 디렉토리에 배치하면 컨테이너가 실행될 때 자동으로 압축 해제 및 컨텍스트 등록을 수행함.
* **통합 개발 환경(IDE) 활용**:
    * **IntelliJ IDEA**, **Eclipse** 등을 사용하여 배포를 간소화함.
    * IntelliJ 트라이얼 기간 만료 시 실행이 제한될 수 있으므로 주의가 필요함.

---

## 2. 웹 설정의 중심: web.xml
자바 웹 애플리케이션의 동작 방식과 경로를 제어하는 배포 서술자(Deployment Descriptor)의 역할입니다.

* **web.xml의 역할**: 웹 컨테이너 구동 시 프로젝트의 환경 설정을 정의하는 핵심 파일임.
* **주요 설정 내용**:
    * **Context 설정**: 웹 프로젝트의 시작 경로를 지정함.
    * **서블릿(Servlet) 매핑**: 특정 URL 요청과 실행될 자바 클래스를 연결함.
* **설정 방식의 진화**: XML 수동 작성 방식에서 자바 코드 내 **어노테이션(@WebServlet)** 활용 방식으로 발전하여 관리가 편해짐.
* **유지보수**: 파일 저장 시 물리적 경로와 논리적 경로가 일치해야 에러가 발생하지 않음.

## 3. 메이븐(Maven)을 통한 라이브러리 관리
복잡한 외부 라이브러리 설치와 버전 관리를 자동화하는 빌드 도구의 핵심 개념입니다.

* **메이븐 저장소(Maven Repository)**: 프로젝트에 필요한 라이브러리들이 모여 있는 곳으로, 거의 모든 자바 라이브러리 관리가 가능함.
* **의존성(Dependency) 관리**:
    * 라이브러리 간의 연쇄적인 의존 관계를 메이븐이 자동으로 파악하여 설치함.
    * 개발자가 일일이 파일을 찾아다닐 필요 없이 `pom.xml` 설정만으로 관리가 끝남.

---

## 4. pom.xml과 프로젝트 구조 최적화
프로젝트 빌드 정보와 환경 설정을 담은 `pom.xml` 활용법 및 표준 구조입니다.

* **pom.xml (Project Object Model)**:
    * 프로젝트 이름, 개발자, 라이선스 등 모든 정보를 객체로 다룸.
    * 자바 버전 명시: `<properties>`에 소스 및 타겟 버전을 지정함.
* **환경 설정 최적화**:
    * **JDK 일치**: 프로젝트 SDK와 컴파일러 버전을 반드시 동일하게 맞춰야 함.
    * **서블릿 API 추가**: 웹 개발을 위해 `javax.servlet-api`를 의존성에 등록함.
* **표준 디렉토리 구조**:
    * `src/main/java`: 자바 소스 코드 위치.
    * `src/main/webapp`: HTML, JSP 등 웹 자원 위치.
