# 오늘 배운 내용

## Java

### 정적 변수와 정적 메서드(정적 멤버들)

1. 정적 멤버들
    - 클래스에서 static 키워드를 붙여 정의한 멤버이다.
    - 정적 멤버들은 class 에 속해 있는 멤버이고, class에 하나씩만 생성된다
    - 객체 외부에서 접근할때는 class 식별자로 접근할 수 있다.
2. 정적 변수
    1. 인스턴스 변수
       - 객체마다 하나씩 있는 변수이다.
       - 객체의 일반적인 필드를 말한다.
       - 객체가 소멸하면 인스턴스 변수도 소멸한다. 
    2. 정적 변수
       - static 키워드를 붙여 정의한 필드들이다.
       - 일반 필드와는 달리 클래스에 하나씩만 생성되며, 모든 인스턴스에서 공유한다.
       - 정적 변수는 클래스 변수라고도 하며, 외부에서 접근할 때는 잠조 변수의 식별자가 아닌, 클래스의 식별자로 접근 해야한다.
       * 객체 외부에서 적적 멤버에 접근
        ```   
        Sample sample = new Sample(10);
        int num = sample.numInstances;
        int num = Sample.numInstances;
3. 정적 메서드
   -  static 키워드를 붙여서 정의한 메서드이다.
   -  정적 메서드는 객체가 상성되지 않아도 클래스 식별자로 호출할 수 있다.
   -  객체가 생성되기 이전이라도 정적 메서드로 정적 변수에 접근할 수 있다.
   -  객체의 일반 필드나 메서드에 접근할 수 없다. 정적 멤버인 정적 변수나 정적 메서드에만 접근할 수 있다.
4. 정적 상수
   - 클래스를 정의하면서 상수를 쓸 수 있다.
   - 상수 필드가 모든 인스턴스에서 같은 값을 가지면, 저장 공간을 절약하기 위해 정적 변수로 선언한다.
  
### 객체 배열

1. 객체 배열
   - 객체 배열의 각각의 원소들은 객체가 아닌 참조 변수이다.
    ```
    Sample[] list= new Sample[3];
    배열은 객체이므로 new 연산자로 생성하고, 배열 참조 변수로 참조한다.
2. 슈퍼 클래스의 객체 배열 활용
    - 슈퍼 클래스의 참조 변수는 슈퍼 클래스의 인스턴스뿐만 아니라 서브 클래스의 인스턴스도 참조할 수 있다.
    - 재정의한 메서드는 동적 바인딩(dynamic binding)을 한다.
    - 그래서 슈퍼 클래스의 객체 배열을 이용하면, 슈퍼 클래스의 인스턴스뿐만 아니라 서브 클래스의 인스턴스들까지 하나로 묶어서 관리할 수 있다.

### 열거형

1. 열거형
   - 어떤 값을 나타내는 여러 상수들의 묶음을 만들기 위한 자료형이다
   - 클래스의 일종이며, enum 키워드를 이용해서 정의한다.
   - 열거형 상수는 내부적으로 정적 상수로 생성된다.
   - 코드가 단순해지며 가독성이 좋아지고, 인스턴스 생성과 상속을 방지한다.
   1. 열거형 변수
      - 열거형은 클래스의 일종이므로, 자료형이 열거형인 참조 변수를 선언해 열거형 상수를 대입한다.
      - 열거형 변수에는 열거형 상수만 대입할 수 있다.
      - 열거형 상수는 정적 상수이므로, 열거형 식별자로 접근한다.
   2. 열거형의 생성자
      - 열거형은 특별한 형태의 클래스이르모, 일반 클래스처럼 생성자(constructor)가 있어야 한다.
        - 물론 생성자를 정의하지 않으면 기본 생성자(default constructor)가 추가된다. 
      - 열거형은 생성자의 접근 제어를 private으로 지정해야한다.
      - 열거형의 생성자를 public이나 protected로 지정하면 컴파일 오류가 발생한다.
        - 열거형은 고정된 상수들의 집합이며, 컴파일 단계에 모든 값을 알고 있어야 한다. 즉 다른 클래스에서 열거형에 접근해서 동적으로 어떤 값을 정해줄 수 없다.
        - 열거형 내에서도 new 연산자로 인스턴스를 생성할 수 없으며, newInstance, clone 메서드 등을 사용할 수 없다. 
   3. 열거형을 문자열로 변환
      - `name` 메서드
        - 열거형 상수의 식별자를 문자열로 반환한다.
      - `toString` 메서드
        - Java 클래스 계층 구조에서 최상위 클래스인 Object 클래스에서 정의된 메서드로, 객체의 문자열 표현을 반환한다.
   4. 열거형 상수의 메서드
      - ordinal 메서드
        - 열거형 상수가 열거형에서 몇 번째 객체인지 반환한다.
      - compareTo 메서드
        - 인자로 주어진 열거형 상수를 기준으로 앞뒤로 몇 번째에 있는지 비교한다.
      - valueOf 메서드
        - 인자로 주어진 문자열과 같은 이름을 가지는 열거형 상수를 반환한다.
         
### 클래스의 정의 절차와 UML

* 클래스의 정의 절차로는 대상 선택 → 속성 설정(필드) → 기능 설정(메서드) 순서이다.
1. UML(Unified Modeling Language; 통합 모델링 언어)
   - 소프트웨어 공학에서 사용되는 표준화된 범요 모델링 언어이다.
* 클래스 다이어그램
  - 작업 하기전 클래스 다이어 그램으로 작업에 필요한 기능등 절차를 짜고 필요한 순서를 클래스 다이어그램으로해서 출력을 하고 작업을 시작하기도 한다.
  - UML,ERD 등을 작성하기 위한 웹사이트, 프로그램등이 있다. 
 
### 상속

1. 상속(Inheritance) 
   - 이전에 작성한 클래스를 이어받아 새로운 클래스를 생성하는 기법이다.
   - 객체 지향 프로그래밍에서 코드를 재사용하기 위한 중요한 기법
   - 기존의 코드를 재활용 하기 위한 방법이다.
   - 상속을 통해 기존 클래스의 필드와 메서드를 다시 사용할 수 있어 코드가 중복되는 것을 줄일 수 있다.
   - 필요에 따라 기존 클래스의 일부분을 변경할 수 있다.
   1. 상속(Is-a)관계
      - 클래스 간에 상속 관계가 성립하려면 Is-a 관계가 성립해야 한다.
      - Is-a 관계는 "A는 B이다"
   2. 슈퍼 클래스(Super class)
      - 상속하는 클래스이고
      - 기초 클래스(Base class; 기반 클래스), 부모 클래스(Parent class)라고도 부른다.
   3. 서브 클래스(Sub class)
      - 상속받는 클래스이다.
      - 유도 클래스(Derived class), 자식 클래스(Child class)라고도 부른다.
    * super 메서드
        - 서브 클래스 생성자에서 super 메서드로 슈퍼 클래스의 생성자를 호출한다.
        - super 메서드는 서브 클래스 생성자의 첫 번째 문장에서만 호출할 수 있다. 즉,
        - 서브 클래스의 필드를 초기화하기 전에 슈퍼 클래스의 필드를 초기화한다.
        - 서브 클래스의 생성자에서 super 메서드로 슈퍼 클래스의 필드를 초기화할 값을 슈퍼클래스의 생성자에 전달한다.
        - 그래서 서브 클래스의 인스턴스를 생성할 때는 슈퍼 클래스의 필드를 초기화할 값도 받아와야 한다. 

### 메서드의 재정의

1. 메서드 오버라이딩(Method overriding; 메서드 재정의)
   - 메서드 오버로딩(메서드 중복 정의)은 매개변수의 자료형이나 매개 변수의 개수가 다르면,
   - 클래스에 식별자가 같은 메서드가 여러개 있을 수 있는 것을 오버로딩이라고 한다.
   - 메서드 오버라이딩(메서드 재정의)은 상속관계에서, 슈퍼 클래스에서 정의한 메서드를 서브 클래스에서 재정의할 수 있는 것이다.
     - 슈퍼 클래스의 메서드와 원형이 같은 메서드를 서브 클래스에서 다시 정의할 수 있다. 
   - 서브 클래스에서 슈퍼 클래스의 메서드를 재정의 할때, @Override 주해(annotation)를 붙여서
   - 컴파일러에게 이 메서드가 재정의한 메서드임을 명시적으로 알린다.
   * 주해
     -  Java 소스 코드에 추가하는 메타데이터의 일종이며, 앞에 @ 기호를 붙여서 사용한다.
     -  주해는 Java 컴파일러에게 추가적인 정보를 전달하며, 필요한 처리를 하도록 한다.
   * @Override 주해
     - 재정의 하는 메서드에 @Override 주해를 붙이면 컴파일러는 이 메서드가 제대로 재정의된 것인지 등을 확인한다.
     - 재정의하는 메서드는 오류를 방지하기 위해 @Override 주해를 붙이는 것이 좋다.
2. 정적 메서드의 재정의
   - 정적 메서드는 객체가 아닌 클래스에 속한 멤버이다.
   - 정적 메서드는 정적 바인딩(static binding)을 하기 때문에, 메서드의 식별자가 슈퍼 클래스 메서드와 같다고 해도 별개의 메서드이지 재정의하는 게 아니다.
   - 따라서 @Override주해를 붙일 수 없다.

### 정적 바인딩과 동적 바인딩

1. 메서드를 호출할때 어떤 메서드를 호출할지 결정하는 방식
   1. 정적 바인딩
      - 컴파일 단계에서, 참조 변수의 자료형에 따라 어떤 메서드를 호출할지 결정한다
      - 일반 메서드와 정적 메서드(클래스 메서드)는 정적 바인딩을 한다.
   2. 동적 바인딩
      - 프로그램 실행 단계에서 참조변수가 참조하는 객체의 자료형에 따라 어떤 메서드를 호출할지 결정한다.
      - 재정의한 일반 메서드(인스턴스 메서드)는 동적 바인딩을 한다.
