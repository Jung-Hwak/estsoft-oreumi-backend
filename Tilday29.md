# 스프링 부트 (Spring Boot) & 프레임워크(Framework)  (2026-01-16)

## 1. 스프링 프레임워크, 왜 쓰는 걸까?
### 1.1 POJO (Plain Old Java Object)
* **개념 :** 특정 기술에 얽매이지 않는 '순수한 자바 객체'
* **가치** 복잡한 기술적인 설정은 스프링한테 맡기고, 우리 개발자는 진짜 중요한 '비즈니스 로직'에만 집중해서 코드를 짤 수 있게 해준다, 덕분에 코드가 깔끔해지고 유지보수도 쉬워져서 사용함

### 1.2 스프링의 3대장 (핵심 기술)
1. **IoC/DI (제어의 역전 / 의존성 주입):** 내가 직접 객체를 만들지(new) 않고, 스프링 컨테이너가 대신 만들어서 나한테 꽂아주는 방식. 부품을 갈아 끼우기 편해져서 결합도가 낮아지는 게 핵심.
2. **AOP (관점 지향 프로그래밍):** 로깅이나 보안처럼 모든 코드에 공통으로 들어가는 기능을 따로 빼서 관리하는 거야. 덕분에 핵심 코드만 깔끔하게 남길 수 있음.
3. **PSA (서비스 추상화):** 데이터베이스나 다른 기술이 바뀌어도 내 코드는 거의 안 바꿔도 되게 중간에서 막을 씌워주는 거야. 리모컨 사용법만 알면 어떤 브랜드 TV든 켤 수 있는 거랑 비슷함

## 2. 스프링 부트(Spring Boot)의 매력
* **한 줄 정의:** 스프링을 더 쉽고 빠르게 쓸 수 있게 다 세팅해놓은 프레임워크.
* **뭐가 좋을까?**
    * **자동 설정:** 귀찮은 설정을 알아서 다 해줘서 바로 개발 시작할 수 있음.
    * **내장 톰캣(Tomcat):** 서버를 따로 설치할 필요 없이 그냥 실행만 하면 됨.
    * **스타터(Starter):** 필요한 라이브러리들을 묶음으로 가져올 수 있어서 관리가 엄청 편해짐

## 3. 스프링 MVC는 어떻게 돌아갈까?
요청이 들어왔을 때 내부에서 움직이는 순서. 구조를 잘 파악해두기!

1. **DispatcherServlet:** 우리 프로젝트의 '문지기'야. 모든 요청을 제일 먼저 받아서 적절한 곳으로 보내줌.
2. **Handler Mapping:** 들어온 URL을 보고 "아, 이건 이 컨트롤러가 처리해야겠네!" 하고 찾아줌.
3. **Controller:** 실제 기능을 실행하고, 어떤 데이터를 보여줄지(Model), 어떤 화면으로 갈지(View) 알려줌.
4. **View Resolver:** 컨트롤러가 알려준 화면 이름을 보고 실제 파일 경로(JSP, Thymeleaf 등)를 찾아냄.
5. **View:** 드디어 완성된 화면을 사용자한테 보여주는 단계.

## 4. 의존성 주입(DI)이랑 롬복(Lombok)
### 4.1 생성자 주입 (중요!)
* 생성자가 딱 하나만 있으면 `@Autowired`를 안 써도 스프링이 알아서 주입. 
* 객체를 만들 때 의존성을 확실히 고정할 수 있어서 훨씬 안전한 방식이다.

### 4.2 롬복(Lombok) 라이브러리 쓰기
* **`@RequiredArgsConstructor`:** `final` 붙은 필드들을 모아서 생성자를 자동으로 만들어줘.
* 생성자 주입이랑 찰떡궁합이라 코드가 진짜 짧아지고 읽기 편해져서 필수로 쓰게 될 것 이다..

---

## 5. 테스트 주도 개발(TDD)이랑 JUnit
### 5.1 테스트 구분하기
* **단위 테스트:** 메서드 하나하나가 잘 돌아가는지 가볍게 확인하는 것.
* **통합 테스트:** `@SpringBootTest`를 써서 전체 스프링을 다 띄우고 실제로 잘 연동되는지 확인하는 것.

### 5.2 JUnit 5로 검증하기
* `assertEquals()`나 `assertTrue()` 같은 메서드로 내가 짠 코드가 예상대로 결과가 나오는지 확인해볼 수 있음.

---

## 6. 로깅(Logging)
* **SLF4J & Logback:** 스프링 부트에서 기본으로 쓰는 로깅 도구야.
* `System.out.println`보다 훨씬 빠르고, 파일로 저장하거나 로그 레벨을 조절할 수 있어서 실무에선 무조건 이걸 써야 함
* **로그 레벨 순서:** `TRACE` < `DEBUG` < `INFO` < `WARN` < `ERROR` (중요도 순서!)

## 7. 실습할 때 꼭 기억할 점
* **폴더 구조:** 소스코드는 `src/main/java`, 설정 파일은 `src/main/resources`에 넣는 게 정석이다.
* **에러 나면 당황하지 말기!**
    1. 프로젝트에 빨간 엑박 뜨면 일단 **[Maven] -> [Update Project]**부터 해보자.
    2. 서버 안 켜지면 콘솔 창 맨 위로 올려서 **"Root Cause"**부터 찾아보는 습관을 들여야함
