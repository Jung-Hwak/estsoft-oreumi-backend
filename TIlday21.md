# 데이터베이스 및 정규 표현식 통합 수업 노트 (2026-01-06)

## 정규 표현식 및 데이터 모델링 기초

### 1.1 정규 표현식 (Regular Expression)
* **개념**: 특정한 규칙을 가진 문자열의 집합을 표현하는 형식 언어.
* **주요 문법**:
    * `[]`: 문자 클래스 (예: `[0-9]` 숫자 중 하나)
    * `*`, `+`, `?`: 반복 기호 (0번 이상, 1번 이상, 0 또는 1번)
    * `^`, `$`: 위치 지정 (시작과 끝)
* **Python 활용**: `re` 모듈의 `match()`, `search()`, `findall()`, `sub()` 메서드 사용.

### 1.2 데이터 모델링 기초 (ERD)
* **피터 첸(Peter Chen) 표기법**: 학술적 용도. 개체(사각형), 속성(타원), 관계(마름모).
* **정보 공학(IE) 표기법**: 실무용. '까마귀 발(Crow's Foot)' 표기법. 사각형 안에 속성을 나열하며 키(Key)와 일반 속성을 구분.

## 관계 데이터 모델 구조 및 특징

### 2.1 관계 모델 용어 정리
| 논리적 모델 (Relational) | 물리적 모델 (DB/Table) | 설명 |
| :--- | :--- | :--- |
| **릴레이션 (Relation)** | **테이블 (Table)** | 데이터의 집합 |
| **튜플 (Tuple)** | **행 (Row)** | 유일한 데이터 한 줄 |
| **속성 (Attribute)** | **열 (Column)** | 데이터의 항목/특징 |

### 2.2 구조적 지표
* **스키마 (Schema)**: 릴레이션의 이름과 구조 (정적).
* **인스턴스 (Instance)**: 실제 저장된 데이터 값 (동적).
* **차수 (Degree)**: 속성(열)의 전체 개수.
* **카디널리티 (Cardinality)**: 튜플(행)의 전체 개수.

## 키(Key)와 무결성 제약조건

### 3.1 키의 종류
1. **슈퍼키**: 유일성은 만족하지만 최소성은 만족하지 않을 수 있는 키.
2. **후보키**: 유일성과 최소성을 모두 만족하는 식별자.
3. **기본키 (PK)**: 후보키 중 선택된 메인 키 (Null 불가, 중복 불가).
4. **외래키 (FK)**: 타 릴레이션의 기본키를 참조하여 관계를 형성하는 키.

### 3.2 무결성 제약조건
* **개체 무결성**: 기본키는 절대 Null일 수 없고 중복될 수 없음.
* **참조 무결성**: 외래키는 참조하는 기본키에 존재하는 값이나 Null만 가질 수 있음.
* **도메인 무결성**: 속성값은 정의된 데이터 타입과 범위를 준수해야 함.

## 관계 대수 (Relational Algebra) 기초

### 4.1 일반 집합 연산
* **합집합(∪), 교집합(∩), 차집합(-)**: 두 릴레이션이 합병 가능(속성수와 도메인 일치)해야 함.
* **카티션 프로덕트(×)**: 두 릴레이션의 모든 가능한 조합 생성.

### 4.2 순수 관계 연산
* **셀렉트 (σ)**: 조건에 맞는 행(튜플)만 선택 (수평적 절단).
* **프로젝트 (π)**: 필요한 열(속성)만 추출 (수직적 절단).

## 관계 대수 심화 및 SQL 매핑

### 5.1 연산의 조합과 조인(Join)
* **복합 연산**: $\pi_{이름}(\sigma_{성적='A'}(학생))$ 처럼 여러 연산자를 중첩하여 정교한 질의 수행.
* **자연 조인 (⋈)**: 공통 속성을 기반으로 테이블을 합쳐 의미 있는 정보를 통합하는 핵심 연산.

### 5.2 SQL과의 연결 및 결론
* **SQL 대응**: 셀렉트는 `WHERE`, 프로젝트는 `SELECT`, 조인은 `JOIN` 문으로 구현됨.
* **설계의 결론**: 데이터 무결성과 독립성을 보장하기 위해 정확한 스키마 설계와 키 설정이 데이터베이스 구축의 시작이자 끝임.
