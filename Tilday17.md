### 추상 클래스와 인터페이스

1. 추상 클래스
    - 상속을 목적으로 정의해놓은 클래스
    - Java day06일 차에 했던 Person과 같은 클래스가 추상 클래스 이다.
    - 추상 메서드를 포함하고 있는 클래스를 말한다. 앞에 abstract 키워드를 붙여준다.
    - 추상 클래스로는 객체를 생성할 수 없다.
2. 추상 메서드
    - 정의 없이 선언만 되어 있는 메서드
    - 메서드의 본체가 없고 abstract 키워드를 앞에 붙여주고 끝에 세미콜론(;)을 붙인다.
3. 인터페이스
   - 클래스 작성에 도움을 줄 목적으로 만듬.
   - 정해진 규칙에 맞게 클래스를 만들도록 표준을 제시한다.
   - 인터페이스의 모든 멤버는 public 이어야 한다.
   - 인터페이스를 잘 사용하면 여러 클래스들을 동일한 규격으로 만들 수 있다.
   - 인터페이스를 구현한 몇몇 클래스중에서 같은 동작을 하는 메서드가 필요한 경우 기본 메서드가 있다.
   - 인터페이스를 상속하는걸 구현한다 라고 한다, 인터페이스를 구현하는 클래스에는 인터페이스 추상 메서드를 작성해야한다.
   - 인터페이스는 여러 객체들이 같은 규격을 갖도록 하기 위해 사용한다.

### 예외 처리

- 프로그램에는 예외라는 오류가 존재한다. 오류란 프로그램이 실행되는 도중에 발생하는 문제 상황을 뜻한다.
- 문법적인 오류는 예외에 포함하지 않는다.
- 치명적인 예외상황을 발생할수 있으니 반드시 try-catch 구문으로 처리해줘야 한다.
* 예외 처리
  - 프로그램이 실행되는 도중에 발생할 수 있는 오류를 대비해서 코드를 작성해 놓는 것이다.
  - 프로그램이 비정상적으로 종료되는걸 막고, 정상적으로 실행이 되는것을 유지하기 위해 사용한다.

1. try-catch 구문
  - try 블록 내에 실행될 코드에서 예외가 발생하면
  - catch 블록에 작성된 문장이 실행이 되고 다음으로 finally 블록이 실행된다.
    ```
    tru {                                                   1. 예외가 발생했을때 순서
        System.out.println("n1 / n2 = " + (n1 / n2));        
    } catch (Exception e) {                                 2. 예외가 생겼을때 catch 구문의 작성된 문장이 실행
        System.out.println("Exception occurs");
    } 
    finally {                                               3. 예외가 있건 없건 마지막 문장이 실행
        System.out.println("Finally block");
    }
2. try-with-resources 구문
   - try 키워드 다음에 괄호가 있으면 자원으로 인지한다.
   - 문장의 끝에서 열린 자원이 자동으로 닫히도록 한다.
   - try 키워드 바로 다음에 괄호가 있으면 자원으로 취급
    ```
    static String readFirstLineFromFile(String path) throws Ioexception {
        try (BufferedReader br = new BufferedReader(new FileReader(path))) {
            return br.readLine();
        }
    }

### Java 표준 라이브러리

- Java 표준 라이브러리에서는 제공하는 패키지가 다양하다.
1. java.lang 패키지 
   - java.lang 패키지는 프로그래밍에 필수적인 클래스들을 포함하고 있으며, import문을 사용하지 않아도 된다.
   1. Object 
      - Object클래스는 java 클래스 계층 구조에서 최상위 클래스이다.
      - object 클래스는 따로 지정하지 않아도 컴파일러가 자동적으로 지정해서 상속하게 된다.
      1. object :`clone()` - a객체 a자신을 복사해서 새로운 객체를 만드는 메서드
      2. boolean : `equals(Object obj)` - obj와 이 객체의 값이 같은지 검사
         - 두 인스턴스가 같은 값을 가지고 있는지 확인하는 메서드이다.
         - Object 클래스의 equals 메서드는 매개 변수가 Object 이다.
      3. void : `finalize()` - 쓰레기 수집기에 의해 호출
         - Garbage collector 에 의해 객체가 소멸되기 직전에 호출이 된다.
         - Object 클래스의 finalize 메서드는 아무것도 하지 않는다.
         - 서브 클래스에서 재정의해 할당받은 시스템 자원을 해제하는 등의 작업을 할 수 도있다.
         - Garbage collector은 언제 호출 될지 모르기때문에 finalize 메서드는 쓰는걸 지양 하는게 좋다.
      4. Class<?> : getClass() - 이 객체를 생성한 클래스의 Class 객체를 반환
         - 어떤 클래스의 정보를 담고 있는 클래스이다.
      5. int : hashCode() - 
         - 객체의 해시 코드를 구해 반환하는 메서드이고, 운영체제가 제공하는 기능이다.
         - equals 메서드를 재정의하면 hashCode 메서드도 재정의해야 한다. 
      6. String : toString() -
* toString 메서드
  - Object 클래스의 toString 메서드는 객체의 문자열 표현을 반환하는 메서드이다.
  - 따로 재정의 하지 않으면 (패키지, 클래스 식별자, 객체의 참조값을 포함하는 문자열)을 반환해준다.
* java.lang.String 클래스
  - char 자료형 배열과 그에 관련된 메서드들이 정의되어 있다.
  - String 클래스의 인스턴스는 내용을 바꿀 수 없다.
### 정적 import문

- 정적 상수나 정적 메서드에 접근할때는 클래스 식별자를 이용해 접근한다.
- 정적 import문을 사용하면 정적 멤버에 접근할때 클래스 식별자를 생략할 수 있다.
